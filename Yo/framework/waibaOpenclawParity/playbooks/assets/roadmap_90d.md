# WAIBA 90 Day Roadmap (Parity and Superiority)

## Sprint 1 (Weeks 1-2): Runtime Execution Power
- Completed: deliver `Shell.*` skill with policy-based command execution.
- Completed: deliver `LLM.*` router with automatic text/vision model selection.
- Completed: deliver `Vision.*` facade (`Describe`, `OCR`, `Detect`).
- Add regression playbooks for shell and LLM-enabled flows.

## Sprint 2 (Weeks 3-4): Perception Layer
- Harden local YOLO adapter packaging and JSON output contract.
- Standardize model path/config strategy in project scope.
- Add visual assertion actions for automation/testing.

## Sprint 3 (Weeks 5-8): Parallelism and Recovery
- Implement bounded worker pool for true parallel branches.
- Add deterministic join semantics and conflict rules.
- Add selector self-healing heuristics in CDP/UIA paths.

## Sprint 4 (Weeks 9-12): Intelligence and Telemetry
- Add planner module for goal -> playbook decomposition.
- Add runtime metrics output for reliability and latency.
- Publish parity benchmark results vs OpenClaw baselines.

## Exit Criteria
- >= 95% parity coverage on selected benchmark capabilities.
- >= 2 areas with WAIBA superiority.
